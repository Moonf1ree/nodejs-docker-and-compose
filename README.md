# Докеризация приложения КупиПодариДай

## Что реализовано

- Бэкенд и фронтенд докеризированы multi-stage сборкой.
- Бэкенд запускается в контейнере через `pm2-runtime`.
- Добавлен `docker-compose.yml` с тремя сервисами: `backend`, `frontend`, `database`.
- Переменные окружения вынесены в `.env` (пример в `.env.example`).
- Для фронтенда добавлен nginx-конфиг с `try_files` для SPA-роутинга.

## Проверка локального запуска

1. Скопируйте `.env.example` в `.env` и заполните актуальными значениями.
2. Запустите проект:
   ```bash
   docker-compose up --build
   ```
3. Проверьте доступность:
   - Frontend: `http://localhost:8081`
   - Backend: `http://localhost:4000`

## Проверка кэширования слоёв Docker

1. Выполните первую сборку:
   ```bash
   docker-compose build
   ```
2. Измените только переменную порта в `.env`.
3. Выполните сборку повторно. Слой установки зависимостей должен быть помечен как `CACHED`.

## Деплой на сервер

1. Установите на сервер Docker и Docker Compose.
2. Склонируйте репозиторий на сервер.
3. Создайте файл `.env` на сервере.
4. Запустите:
   ```bash
   docker-compose up -d --build
   ```
5. Проверьте логи:
   ```bash
   docker-compose logs -f
   ```
